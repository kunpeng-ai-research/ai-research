## 引言：为什么你需要一个OpenClaw？

你有没有想过：一个AI助手，完全由你控制，想怎么用就怎么用，数据绝不外泄？

OpenClaw就是这样的存在。它不是ChatGPT、不是Claude、不是文心一言，而是一个**你可以在自己电脑、服务器甚至树莓派上运行的AI网关**。

### OpenClaw是什么？

OpenClaw是一个自托管的网关（gateway），它连接你已经在用的聊天应用（WhatsApp、Telegram、Discord、微信、iMessage等）和AI模型（OpenAI GPT、Anthropic Claude、Google Gemini、本地Ollama等）。

简单说：**你用熟悉的聊天App发消息，OpenClaw收到后调用AI，再把AI的回复发回给你。** 整个过程完全在你控制的环境中进行。

### 为什么选OpenClaw而不是直接用ChatGPT？

| 对比维度 | OpenClaw | ChatGPT/Claude云端版 |
|---------|---------|---------------------|
| **数据隐私** | ✅ 完全自控，数据不出你的设备 | ❌ 数据在第三方服务器 |
| **多平台** | ✅ 20+平台同时在线 | ❌ 只能通过官方App或网页 |
| **模型选择** | ✅ 任何provider都能接 | ❌ 只能使用 provider 自家模型 |
| **成本** | ✅ 一次部署，持续使用 | ❌ 按月订阅或按token付费 |
| **控制力** | ✅ 完全可定制、可扩展 | ❌ 功能固定，不能改 |
| **可用性** | ✅ 24/7在线，不需要外网 | ❌ 依赖 provider 可用性 |

## 第一部分：快速安装（5分钟）

### macOS / Linux 安装

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

### Windows 安装

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### 验证安装

```bash
openclaw --version
```

---

## 第二部分：Onboarding - 初始配置

```bash
openclaw onboard --install-daemon
```

按向导完成 AI provider 配置、Gateway Token 生成，约2分钟。

---

## 第三部分：连接 Telegram

1. Telegram 搜索 @BotFather，发送 `/newbot` 获取 Bot Token
2. 编辑 OpenClaw 配置：

```bash
openclaw config edit
```

```json5
{
  plugins: {
    telegram: {
      botToken: "你的Bot Token"
    }
  }
}
```

3. 重启：`openclaw restart`
4. 在 Telegram 给机器人发消息，收到 AI 回复即成功！
